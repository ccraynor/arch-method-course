===========================================================
DESIGN INTENT
===========================================================

The ARCH Method course will use a dual-scenario instructional architecture.

1. Longitudinal Anchor Case
   Meridian Regional Health Network remains the single continuous course-design case used for:
   • Worked examples
   • Guided practice
   • Decision points
   • Governance checkpoints
   • Performance tasks
   • Cumulative artifact development
   • Module transitions

2. Spotlight Scenario System
   Each lesson introduces a short authentic instructional design scenario that demonstrates the transferability of the ARCH Method across sectors.

Spotlight scenarios function as contrast cases.

They are instructional examples—not replacement performance tasks—and should reinforce the current ARCH workflow while preserving continuity of the Meridian anchor case.

===========================================================
GLOBAL DESIGN PRINCIPLES
===========================================================

Every spotlight scenario shall:

- Represent an authentic instructional design problem.
- Align directly with the ARCH phase and lesson objective.
- Demonstrate why the current ARCH step matters.
- Require professional judgment rather than recall.
- End with an authentic design decision.
- Illustrate transfer of the ARCH workflow to a different organizational context.
- Be concise enough to support a 3–5 screen instructional segment.

Scenario variation exists to improve transfer—not simply to increase variety.

===========================================================
OPTIONAL MICRO-STORYLINE CONTINUITY
(RECOMMENDED)
===========================================================

To strengthen narrative cohesion without replacing the Meridian anchor case, selected spotlight scenarios may reappear across modules as organizational "snapshots."

Example progression:

Module 1
State Education Agency defines scope.

↓

Module 2
The agency sequences statewide professional learning.

↓

Module 3
A new state policy creates alignment conflicts requiring architectural revision.

↓

Module 4
The completed architecture is presented to the State Quality Assurance Board for approval.

This same progression may be used for the Community College, Corporate, Healthcare, or Nonprofit organizations.

These recurring snapshots should be brief (1–2 screens) and demonstrate that instructional design is an iterative organizational process rather than a collection of isolated tasks.

===========================================================
BUILD GUIDELINES
===========================================================

- Meridian Regional Health Network remains the only cumulative performance-task scenario.
- Spotlight scenarios should remain concise (3–5 instructional screens).
- Every spotlight scenario should culminate in one authentic instructional design decision.
- Scenario complexity should increase across modules through governance, stakeholder conflict, executive responsibility, and decision defense—not through unnecessary domain complexity.
- Professional responsibility should evolve across the course from building, to engineering, to leading, to defending.
- Maintain clear traceability between each spotlight scenario, the corresponding ARCH phase, lesson objective, and cumulative performance task.
