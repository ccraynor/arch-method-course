Lesson 1.4b

Apply Steps 7 and 8: Optimize Components and Audit the Trace

Activity Context

Purpose

Apply the two-question optimization test (Step 7) and a forward scope trace (Step 8) to a new set of MRHN components -- separate from those modeled in Screens 2 and 3. This practice transfers both skills from observed expert execution to independent application.

Instructions

  1. Part 1: Apply the teachability and assessability questions to each component. Select Split, Merge, or Confirm. If you select Split or Merge, describe the revised component(s).
  2. Part 2: Review each component's listed parent SI. Flag any misattribution and select the correct parent.
  3. Select Check My Work to see expert analysis for all items.

Success Criteria

  • Each verdict (Split / Merge / Confirm) matches the expert for the right reason -- not by coincidence
  • Revised descriptions preserve the original instructional obligation while correcting the granularity problem
  • The misattributed component in Part 2 is correctly identified and its parent reassigned

Expected Evidence

Three optimization verdicts with revised descriptions where applicable; six scope trace judgments with one corrected parent attribution. Expert reasoning is shown for all nine items after submission regardless of your answers.

Part 1 -- Unit Optimization

Apply the two-question test to each component. Q1 (Teachability): Can this be taught as one instructional unit without losing coherence? Q2 (Assessability): Can this be assessed by one well-formed item? If either question fails, the component needs to be split or merged.

DC-47 SI-16 • Bucket 4 Analysis

Conduct the MRHN post-incident after-action review and produce the formal lessons learned report, distinguishing operational observations from governance-required findings subject to legal review under RC-4.

Optimization Test

Q1 -- Teachability: Can "conduct the after-action review" (facilitated group analysis) and "produce the formal lessons learned report" (structured document production with RC-4 requirements) be taught as a single instructional unit?

Q2 -- Assessability: Can one well-formed item assess both the review facilitation behavior and the RC-4 report production behavior at the required depth?

Your verdict for DC-47
DC-40 SI-15 • Bucket 4 Application

Locate the MRHN system restoration criteria document within RC-3 and identify the three technical thresholds that must be met before restored systems are cleared for clinical use.

Optimization Test

Q1 -- Teachability: Is locating a document and naming thresholds from it a coherent instructional unit, or is it a lookup behavior that only becomes meaningful within a larger evaluative act?

Q2 -- Assessability: Can locating and naming thresholds from RC-3 be assessed by one well-formed item that reflects the Application-level demand of SI-15 (evaluating restored systems against criteria)?

Adjacent Component DC-41: Evaluate restored MRHN systems against the RC-3 technical thresholds to determine restoration clearance for clinical use.
