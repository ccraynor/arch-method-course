Module Overview

Module 1: Bound the Course

Phase 1: Constraint Identification and Scope Architecture

In this module you turn the Meridian project brief into a defensible course architecture. You will identify the constraints that govern the design. Then you organize the scope, calibrate it, and decompose it into teachable components.

Six Lessons

  1. Lesson 1: Meridian Project Intake Est. 45-60 min
  2. Lesson 2: Constraint Identification and Documentation Est. 90-120 min
  3. Lesson 3: Bucket Mapping and Scope Organization Est. 90-120 min
  4. Lesson 4: Domain Calibration Est. 60-90 min
  5. Lesson 5: Performance Arc and Decomposition Est. 90-120 min
  6. Lesson 6: Unit Optimization and Scope Trace Audit Est. 90-120 min

Five Artifacts You Will Produce

MRHN_BucketMap_v1: A structured map organizing all 16 scope items into four workflow-based buckets with documented rationale.

MRHN_CalibratedScopeMap_v1: The bucket map with domain authenticity and taxonomy level annotations applied to every scope item.

MRHN_PerformanceArc_v1: A validated arc statement confirming the course sequence produces the required end-of-training performance.

MRHN_DecompositionMap_v1: All 16 scope items broken into teachable, assessable components with governance documentation.

MRHN_DecompositionMap_v2: The optimized decomposition map with all split, merge, and correction decisions applied.

Estimated total module time: 8-10 hours

You finished the credential introduction and wrote your commitment note. You are ready to start designing the Meridian course architecture.

Where this leads

In Module 2 you use this architecture as the foundation for engineering the learning sequence.
