Before You Begin

What is the difference between a binding constraint and a negotiable constraint?

Saved

Reference answer

A binding constraint is a non-negotiable condition that must govern every design decision. A negotiable constraint is a preference or default that can be adjusted with stakeholder agreement.

3 Complete the Bucket Map

Step 3 produces the formal artifact that closes the bucketing phase. The balance-checked bucket map becomes a governed document: all 16 scope items assigned, all four buckets confirmed, and the five-field governance record attached.

The completed artifact has three components:

  • Scope item table: All 16 SI items with bucket assignment and workflow rationale for each
  • Balance check summary: Four-criterion verdict for each bucket
  • Governance record MRHN-D-002: Decision statement, constraint reference, rationale, risk note, and checkpoint linkage
4 Domain Authenticity and Taxonomy Calibration

Step 4 applies two calibration lenses to the completed bucket map before decomposition begins. Both lenses can produce adjustments to scope items: splits, consolidations, or calibration annotations.

  • Domain Authenticity: Does each scope item reflect how healthcare IT professionals actually perform incident response under HIPAA and IR-2024 constraints, not an idealized or generic version of the task?
  • Taxonomy Calibration: Is the cognitive demand of each scope item, recall, application, analysis, or evaluation, appropriate for the MRHN learner population within the project constraint set?

Why Calibration Precedes Decomposition

Decomposition in Module 2 breaks each scope item into its component learning tasks. If a scope item is pitched at the wrong taxonomy level or does not reflect real domain practice, those errors compound. Every decomposed component inherits the original miscalibration. Step 4 is a pre-decomposition check. It is your last chance to adjust scope before the architecture becomes too granular to restructure efficiently. A scope item adjusted here costs one governance record. The same adjustment made after decomposition costs a re-design of multiple components.
