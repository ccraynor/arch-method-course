Lesson 1.4b

Unit Optimization and Scope Trace Audit

Phase 2 -- Step 7: Optimize for Unit Structure Phase 2 -- Step 8: Scope Trace Audit

MRHN_DecompositionMap_v1 is on record with all 48 DC components mapped across 16 scope items. Before this map can drive Module 2 instruction design, two quality checks must pass. Step 7 tests whether each component is the right size for independent teaching and assessment. Step 8 audits the full map for scope integrity -- every component must trace to a parent scope item, every scope item must generate at least one component, and no component may exist outside LC-1 scope boundaries. Together, Steps 7 and 8 produce MRHN_DecompositionMap_v2: the validated structural input that Module 2 will use to engineer instruction.

What Steps 7 and 8 Do

7 Phase 2 Optimize for Unit Structure Step 7

"Is each DC component the right size -- specific enough to teach independently, bounded enough to assess cleanly?"

Step 7 applies the unit optimization criteria to every component in MRHN_DecompositionMap_v1. A component fails the optimization check if it is too large -- covering multiple distinct skills that cannot be assessed by a single item -- or too small, fragmenting a coherent skill into sub-steps that have no instructional value in isolation.

Optimization decisions are not about preference. Each check is governed by two questions: can this component be taught in a single instructional unit, and can learner competence on this component be assessed by a single, well-formed item? Components that fail either check are split, merged, or redescribed before Step 8 begins.

Interim output: MRHN_DecompositionMap_v1 (optimized draft)
8 Phase 2 Scope Trace Audit Step 8

"Does every component trace to a scope item? Does every scope item have at least one component? Are there orphans?"

Step 8 audits the optimized decomposition map against the scope registry. The audit runs three checks: a forward trace (every SI item in the scope registry generates at least one DC component), a backward trace (every DC component maps to exactly one parent SI item), and an orphan check (no component exists without a valid parent, and no parent SI item is unrepresented in the map).

The Scope Trace Audit is the final gate before Module 2. A failed audit -- any DC component that cannot be traced to an SI item in the scope registry -- means something outside LC-1 scope has entered the map, or a scope item was silently dropped during decomposition. Either condition must be resolved before Module 2 engineering begins.

Produces: MRHN_DecompositionMap_v2

Why Optimization Precedes the Audit

The Scope Trace Audit runs against whatever is in the map at the time of the audit. If you audit before optimizing, you may pass the trace check on a component that will later be split -- producing two components, one of which may not trace clearly to the original parent SI item. Running Step 7 first means the map being audited is the map that will actually go to Module 2.

Optimization also surfaces a structural problem the audit cannot catch: components that are technically traceable but instructionally incoherent. A component that traces to SI-09 but bundles three distinct analysis skills will pass the audit and fail in Module 2, when the design team discovers the component cannot be sequenced or assessed as a unit. Step 7 catches that problem before the audit locks the map.
