Lesson 1.1b

Applying Design Document Synthesis

Activity Context

Purpose

Seeing an expert extract constraints is not the same as extracting them yourself. This practice builds the systematic reading habit that makes constraint identification reliable rather than a matter of what happens to be noticed.

Instructions

  1. Return to Screen 2 to review the Meridian brief if needed.
  2. Complete all four constraint type fields and the binding identification field below.
  3. Extract at least six constraints total across all four types.
  4. Select Check My Work when ready.

Success Criteria

  • At least one constraint extracted in each of the four type fields
  • At least six constraints identified in total across all fields
  • Binding identification names specific constraints with reasons, not just a category
  • Performance field references IR-2024 and role authority specifically

Expected Evidence

A four-part constraint extraction with a binding versus negotiable ranking precise enough to use as the source material for the MRHN Bucket Map in Screen 5.

Source document: The Meridian project brief is on Screen 2 of Lesson 1.1a (m1-l1a-s2.html). Read it systematically -- all four constraint types are present but they are not all labeled as constraints in the brief.

Constraint Type 1

What must learners be able to do, and with what precision? Look in the Required Performance section. List each constraint on its own line.

Constraint Type 2

What legal or compliance requirements apply? These have authority external to the client -- neither the client nor the designer can negotiate them away. Check the Background and Regulatory sections.

Constraint Type 3

What conditions bound what is buildable within this project? Include time, budget, format, SME access, and available source materials. Check the Project Constraints section.

Constraint Type 4

What client-stated conditions govern process or quality? These are not externally mandated but are non-negotiable within this project without a documented conversation. Check the Course Request and Deliverable Request sections.

Name at least three specific constraints from the fields above and explain what makes each one binding -- that is, what would have to change for it to flex.
